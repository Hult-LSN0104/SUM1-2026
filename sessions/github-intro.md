# GitHub in 5 Minutes

You don't need to know much about GitHub to succeed in this course. You need to know three things. That's it.

## 1. GitHub is where your code lives

Think of GitHub like Google Drive, but for code. Your course files live on GitHub in a "repository" (or "repo"). You have your own copy of the course repo — that's where you'll do all your work.

## 2. Codespaces is where you edit your code

You don't install anything on your laptop. You click "Open in Codespaces" and a full coding environment loads in your browser — already set up, ready to go.

Your Codespace is connected to your GitHub repo. Code you write in Codespaces gets saved back to GitHub.

## 3. You need to "commit and push" to save your work

Writing code in Codespaces isn't enough — you need to save it back to GitHub so you don't lose it. This takes two steps:

- **Commit** = "I'm saving a snapshot of my work with a note about what I did"
- **Push** = "Send my saved snapshot up to GitHub"

In Codespaces, there's a Source Control icon on the left sidebar (looks like a branching line). Click it, type a short message (like "finished exercise 2"), click **Commit**, then click **Sync Changes**. Done.

We'll practice this in class.

## That's It

There's a lot more to GitHub — branches, pull requests, forks, issues — but you don't need any of that for this course. If you're curious, check out the [full GitHub reference](../../reference/github-basics.md) in this repo.
