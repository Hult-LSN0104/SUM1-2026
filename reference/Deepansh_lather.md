## What I Learned
I learned that GitHub is a platform for storing and sharing code. You can track changes, work with others, and keep a history of everything you edit. Forking lets you copy someone else's repository and make changes without affecting the original. A pull request is how you send those changes back for review. I also learned basic markdown formatting like headings, bold, and italics. The most confusing part was understanding the difference between a fork and a branch, and when to use each one. I am also still getting used to how commits and push work together.

## Still Confused About
